# Day 1 Backend Notes

## What is an API?
An API allows two systems to communicate. Example: Client asks for user data → server returns it.

Request -> Server -> Response

request contains :
- Methods (like GET, POST, PUT, DELETE)
- URL
- Headers
- Body (sometimes)

response contains :
- Status codes
- Body (data)

## HTTP Methods
- GET: retrieve data  
- POST: create something new  
- PUT: update data  
- DELETE: remove data  

## Status Codes
200 → success  
201 → created  
400 → bad request  
404 → not found  
500 → server error
